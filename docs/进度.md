### 接真引擎 + 多步早停修复（已完成，真实跑出）
- **接真引擎评测**：W4A16 Qwen3-14B（单卡 vLLM）跑 19 任务，结果已写入 `README.md` 主要结论与 `docs/eval.md` 第 5 节（定性）。
- **多步早停/幻觉修复**：抓失败轨迹定位真根因 = SFT+量化后 `<think>` 空、第二跳**直接编造结果**而非调工具（非框架 bug）。在 `agent/prompts.py`（A·多步执行纪律）+ `agent/graph.py`（B·`reflect` 反幻觉自检兜底 + `should_continue` 三重门槛：已调过工具才触发 / 限强度 / 反螺旋上限）落地修复。
- **同端点 before→after 实测**（`scripts/eval_ablation.py` 一次出两档）：轨迹成功率明显提升、多步任务完成数提升、relevance 判对率提升到满分；**代价**：工具精确率明显下降（兜底引入额外调用）。**边界**：编排只缓解早停，救不了模型层面的选错工具 / 不敢调写操作 / 长链编造（残余任务属此类）。
- 新增脚本：`scripts/eval_ablation.py`（before/after 两档）、`scripts/eval_subset.py`（子集快测）、`scripts/debug_trace.py`（打印完整消息序列定位失败）。

### MCP 集成（已完成，真实跑通）
- **工具经 MCP 协议对外/被调**：`edu_agent/mcp/server.py` 用官方 `mcp` SDK 把 registry 的 16 个工具暴露为 MCP server（stdio 传输）——list_tools 直接复用现成 OpenAI schema（`parameters`→`inputSchema`）、call_tool 复用 `registry.dispatch` 执行，**工具逻辑零重写**。
- **MCPToolProvider**（`edu_agent/mcp/client.py`）：与 registry **同契约**（`openai_tools()` / `dispatch()`），内部用后台事件循环 + `ClientSession` 经 MCP 协议往返，故同步的 LangGraph 图无需改成 async。`build_agent` / `run_agent` 新增 `tools_provider` 参数（默认本地 registry，传 provider 即走 MCP），`EDU_AGENT_TOOLSOURCE=local/mcp` 工厂切换。
- **验证**：`scripts/mcp_demo.py` 用同一离线 mock 大脑跑通五工具多步任务，每步经 MCP 协议往返；`tests/test_mcp.py` 断言「MCP 工具清单 / 单次结果 / 多步轨迹」与本地直调**逐一致**。全套 **32/32** 通过、ruff 全过。

### 环境
- 数据+工具层：系统 Python（stdlib），`python tests/test_tools.py` 即可。
- Agent 层：`uv sync` 建 3.12 venv（langgraph 1.2.6 / openai 2.43 / pytest）；`uv run python -m pytest tests/ -q` → **32/32**（工具 18 + Agent 编排 3 + 评测框架 7 + MCP 集成 4）。lint：`uv run ruff check` 全过。MCP 集成需可选依赖 `mcp`（`uv pip install mcp` 或 `uv sync --extra mcp`）。

### 关键设计取舍
- **数据+工具层零依赖**：仅 `sqlite3` + 纯 Python；`python tests/test_tools.py` / `python scripts/demo_trajectory.py` 无需 pip 安装、无需联网即可验证。
- **数据内部一致**：学生隐变量 ability × 题目 difficulty → 作答；成绩/错题/知识点掌握度/学习路径四者互相印证（demo 中薄弱生被推的练习点正是其 0 分知识点）。
- **引擎/KG 后端均可替换**：KG 现为内存图（保留等价 Cypher 提示），后续可换真实 Neo4j；引擎层下一步接入。
